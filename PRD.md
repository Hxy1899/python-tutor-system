## 1. **需求背景**

### 目标

构建一个适配中小学Python编程教学场景的智能化编程作业纠错与提示工具。系统能够精确识别常见编程错误，提供引导式提示，帮助学生自主调试与修正代码，降低初学者的编程入门门槛。

### 系统核心目标

- **错误识别与归因**：通过机器学习模型和静态、动态分析手段，识别并诊断学生代码中的错误。
- **引导式提示**：根据错误类型，提供有针对性的提示，引导学生发现并修正错误，而非直接给出答案。
- **Web端交互**：为学生提供在线Python代码编辑器，帮助学生编写、提交、纠错代码。
- **教师辅助功能**：帮助教师高效批改作业、统计学情，减轻工作负担。

------

## 2. **需求概述**

### 系统架构

系统采用三层B/S架构，分为：

1. **前端交互层**：学生端与教师端模块。
   - 学生端：提供在线Python编辑器和错误提示展示功能。
   - 教师端：提供作业管理与学情统计功能。
2. **核心服务层**：处理代码预处理、错误诊断、提示生成等核心业务逻辑。
3. **数据持久层**：数据存储与管理，支持所有操作的数据支撑。

### 核心工作流程

系统核心工作流程图如下图所示：
![核心工作流程](D:\01computer\GraduationDesign\核心工作流程.png)

1. **学生提交代码**：学生通过Web端提交代码。
2. **代码预处理**：系统进行代码规整，过滤干扰项与潜在风险。
3. **错误诊断**：通过静态分析与动态测试，识别代码中的错误并生成结构化诊断结果。
4. **提示生成与反馈**：根据错误诊断结果生成引导式提示并反馈给学生。
5. **作业修改与提交**：学生根据提示修改代码并重新提交，直到完成作业。

## 3. **优先级划分**

### 高优先级需求

- **错误识别与诊断**：确保系统能识别高频常见编程错误并准确反馈。
- **引导式提示生成**：为学生提供具体且易于理解的提示，帮助他们自主调试。
- **Web端编辑器**：提供简洁易用的在线Python编辑器，保证用户体验。

### 中优先级需求

- **教师端功能**：为教师提供作业管理、学情统计等轻量化工具。
- **数据存储与分析**：确保系统能高效存储学生作业数据并进行学习情况分析。

## 4. **需求详情**

### 4.1 交互逻辑

#### 学生端交互

1. 学生通过在线编辑器编写Python代码。
2. 提交后，系统进行预处理与错误诊断。
3. 系统返回错误类型及引导式提示，学生根据提示修改代码。
4. 修改后学生可重新提交代码，直到完成作业。

#### 教师端交互

1. 教师查看学生提交的作业，了解学生的学习进度。
2. 提供轻量化的作业批改与学情分析工具，帮助教师高效管理教学。

### 4.2 功能详情

#### 学生端功能

| 模块       | 功能               | 具体内容                         |
| ---------- | ------------------ | -------------------------------- |
| 代码编辑器 | 提供Python编程环境 | 实时代码编写与编辑，支持语法高亮 |
| 错误诊断   | 提供错误分析与反馈 | 基于错误类型提供引导式提示       |
| 提交与修改 | 提交作业与修正代码 | 学生可根据反馈修改代码，重新提交 |

#### 教师端功能

| 模块     | 功能                   | 具体内容                           |
| -------- | ---------------------- | ---------------------------------- |
| 作业管理 | 管理学生作业提交       | 查看、批改、评分学生作业           |
| 学情统计 | 统计学生学习情况       | 自动生成学生学习进度与错误统计报表 |
| 提示生成 | 自动生成学情反馈与报告 | 根据学生提交的作业反馈分析报告     |

### 4.3 埋点需求

- **用户行为分析**：监控学生提交代码、修改错误的行为，以便更好地优化提示生成与错误诊断。

------

## 5. **关键验收点**

1. **错误诊断准确性**：系统能够准确识别中小学生常见的编程错误，并提供有效反馈。
2. **引导式提示**：学生能够通过系统的提示自我修正错误，且提示内容清晰、易懂。
3. **Web端操作流畅性**：学生和教师端均能实现流畅的操作，页面响应迅速。
4. **数据分析报告**：教师端能够生成准确的学情统计报表，帮助教师掌握学生学习情况。

## 6. **技术选型与架构**

系统总体架构如下图所示：

![系统总体架构](D:\01computer\GraduationDesign\系统总体架构.png)

### 前端

- **框架**：Vue 3
- **编辑器**：CodeMirror 6（用于Python代码编辑与错误提示展示）

### 后端

- **框架**：Python + FastAPI
- **错误诊断**：静态分析工具（pylint、flake8），动态测试（Docker、pytest）
- **数据存储**：MySQL（作业数据存储），Redis（缓存）
- **机器学习**：集成随机森林分类器，使用TF-IDF与AST路径进行错误模式识别与预测

### 安全与隔离

- **代码沙箱**：Docker容器，确保学生代码执行安全，避免潜在恶意代码问题。
